# План работ: Оптимизация и рефакторинг фильтра по тегам

## 1. Описание задачи
В проектах Azure DevOps может быть создано огромное количество тегов. Вывод их всех в виде горизонтальной ленты чип-фильтров перегружает интерфейс и замедляет рендеринг. Задача состоит в том, чтобы сделать панель тегов компактной, отображая только самые популярные теги из загруженных рабочих элементов (Context-Aware) и выбранные фильтры, а для поиска остальных тегов предоставить удобный выпадающий список (Combobox / Search Input).

## 3. План верификации

### Ручное тестирование
1. **Базовое отображение**:
   - Открыть проект с большим количеством задач и тегов.
   - Открыть панель фильтров. Убедиться, что в строке "Tags" отображается не весь список из сотен тегов, а компактный набор из максимум 15 самых частых тегов + поле ввода.
2. **Поиск и добавление тегов**:
   - Ввести в поле поиска тегов имя редкого тега (которого нет в топ-15).
   - Убедиться, что он автодополняется из datalist. Выбрать его.
   - Проверить, что тег появляется в списке чипов с подсветкой (активный фильтр), а список задач фильтруется по нему.
3. **Сброс фильтра**:
   - Нажать на чип выбранного тега для циклического изменения состояния (включение -> исключение -> сброс). Убедиться, что после сброса редкий тег исчезает из видимой строки чипов, освобождая пространство.
