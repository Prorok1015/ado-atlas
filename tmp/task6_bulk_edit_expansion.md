# План работ: Расширение функционала формы массового редактирования (Bulk Edit)

## 1. Описание задачи
Текущий режим массового редактирования (Bulk Edit) в приложении ограничен простым затиранием скалярных полей (status, sprint, assignee, priority) с выполнением последовательных запросов для каждой задачи индивидуально. Для расширения возможностей необходимо:
1. Добавить операции точечной манипуляции тегами (добавление или удаление конкретного тега без стирания остальных).
2. Реализовать массовое изменение родительской связи (Parent) для выбранных задач.(это уже можно сделать перетаскиванием в древовиднов виде, но надо добавить и в балк форму)
3. Оптимизировать отправку данных, упаковывая запросы в один батч-запрос через эндпоинт Azure DevOps `$batch` вместо множественных конкурентных HTTP-вызовов.
4. В балк форме надо поменять поле выбора ассайни, либо переиспользовать tag-search либо использовать кард пикер.
5. Возможно переиспользовать tag-search для всех комбо боксов в проекте(во всяком случае подумать над этим)
6. Подумать какие еще поля можно было бы задавать в балк форме. Может даты начала и таргет или дью дейт
7. Подумать над внешеим видом балк формы, может можно сделать чтото интереснее и удобнее? Может сделать кастомный лайаут для балк формы тоже?

## 3. План верификации

### Тестирование сети
1. Открыть панель Network в Chrome DevTools.
2. Выбрать в дереве 10 задач, на панели массового редактирования выбрать изменение приоритета на P2.
3. Нажать кнопку применения. Убедиться, что на сервер ушел ровно **один** HTTP POST-запрос к эндпоинту `$batch` вместо 10 отдельных PATCH-запросов.

### Ручное тестирование манипуляций тегами
1. Создать три задачи. Присвоить первой теги `dev; ui`, второй — `ui; test`, третьей — `dev`.
2. Выделить все три задачи. Выбрать массовое действие "Add Tag" -> ввести `release`.
3. Применить изменения. Проверить, что:
   - Первая задача теперь имеет теги: `dev; ui; release`.
   - Вторая: `ui; test; release`.
   - Третья: `dev; release`.
   - Никакие старые теги не были затерты.
4. Выделить эти же задачи, выбрать "Remove Tag" -> ввести `ui`.
5. Убедиться, что тег `ui` был удален у первой и второй задачи, а третья осталась без изменений.
