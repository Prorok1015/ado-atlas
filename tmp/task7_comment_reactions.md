# План работ: Интеграция реакций на комментарии

## 1. Описание задачи
Современные системы совместной работы поддерживают быстрые реакции на комментарии в виде эмодзи (лайки, сердечки и др.). В Azure DevOps Comments API есть поддержка реакций (like, dislike, heart, sad, smile, angry). На данный момент приложение выводит комментарии как простой текст без интерактивных элементов. Задача состоит в том, чтобы перейти на использование полноценного Comments API, добавить отображение существующих реакций под комментариями в панели активности сайдбара и реализовать возможность ставить/снимать реакции с оптимистичным обновлением интерфейса.

## 2. Предлагаемые изменения по файлам

### [api.js](file:///c:/Users/proro/source/repos/Prorok1015/easy_ady_extention/api.js)
1. **Расширение формата комментариев в `comments(wid)`** (строка 901):
   - Запросить детальные комментарии, включая реакции. В ADO по умолчанию эндпоинт `_apis/wit/workItems/{id}/comments?api-version=7.1-preview.3` возвращает объект комментария со списком реакций.
   - Маппить массив реакций `reactions` для каждого комментария:
     - Тип реакции (например, `like`, `heart`, `smile`).
     - Счетчик реакций каждого типа.
     - Список пользователей, оставивших реакцию (для определения, ставил ли реакцию текущий пользователь).
     - ID комментария (`id` на уровне API), необходимый для отправки PATCH/PUT-запросов.
2. **Добавление методов управления реакциями**:
   - **`addCommentReaction(wid, commentId, reactionType)`**:
     - Отправляет HTTP-метод `PUT` к эндпоинту:
       `https://dev.azure.com/{org}/{project}/_apis/wit/workItems/{wid}/comments/{commentId}/reactions/{reactionType}?api-version=7.1-preview.3`
   - **`removeCommentReaction(wid, commentId, reactionType)`**:
     - Отправляет HTTP-метод `DELETE` к тому же эндпоинту.
