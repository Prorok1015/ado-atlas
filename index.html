Connect to Azure DevOps

Talk to Azure DevOps with a Personal Access Token. Create one at dev.azure.com → User settings → Personal access tokens with at least Work Items (Read & Write) and Project and Team (Read) scopes. The PAT is stored in chrome.storage.local on this machine only.

Paste a PAT, then fill in your Organization and Project (both are in your dev.azure.com/<org>/<project> URL). The project list fills in automatically once the org is set.

Sign in with your Microsoft (Entra ID) account — no token to copy, and it refreshes automatically. A one-time app registration is required; see the setup guide.

⚠ Work/school (Microsoft Entra) accounts only. Personal Microsoft accounts (outlook/hotmail/live), including a personal account that owns an Azure DevOps org, cannot use OAuth — Azure DevOps has no OAuth resource for them. Use the Token tab (PAT) instead; it works with any account.
